# 0. 설계 동기

나는 그동안 여러 셸을 접해보았다. 실제로 사용한 것은 아니었지만, 배경 지식과 문서를 통해 셸의 설계 원칙과 구동 환경, 핵심 철학 등을 알게 되었다. 그리고 나는 생각했다. 현재 존재하는 셸 언어들은 모두 양 극단에 가있다. 예를 들어 Bash는 사람이 읽기에는 불친절하며, PowerShell은 너무 장황했다. 그 외의 대부분의 셸들을 Bash의 파생형으로 같은 단점을 공유하거나, 이렇다할 장점을 보이지 못했다. 

그래서 나는 마음을 먹었다. 한번 사람과 기계 모두가 읽기 편하면서도, 빠르고 유용한 셸 스크립팅 언어를 만들어보자. 이 프로젝트는 이렇게 시작되었다. 

실제로 만들려는 것이 아니다. 실제로 구현하려는 생각도 없다. 중학교 2학년이 아무리 날고 기어도 셸 스크립팅 언어를 만들기란 하늘의 별 따기이고, 그것을 이미 여러번 겪어봤다. 그러나 단순한 취미로 만드는 것도 아니다. 누군가에게 보여주기 위한 것도 아니다. 이 언어를 설계하면서 나라는 사람에게 줄 수 있는 지식과 성장, 그리고 설계 과정 속에서의 한계를 알아보고 싶었다. 

이 언어 속에는 내가 지난 몇 년 간 가꿔온 모든 지식, 모든 기술이 포함되있다. 이 언어의 설계는 나에 대한 시험이자, 최종 목표의 첫 걸음이다. 과연 나라는 사람이 어디까지 디테일하게 설계했는지, 어디까지 고려했는지, 어디까지 생각했는지를 알아봐주기를 바란다. 

NatureShell이라는 언어를 통해서.
